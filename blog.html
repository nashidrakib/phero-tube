Discuss the scope of var, let, and const.

In JavaScript, we have three ways to declare variables: var, let, and const. var is an older way, known for its function or global scope, and it allows you to change the value of the variable after it's declared. let provides a more controlled approach with block-level scope, making it better suited for loops and if statements, and it can be reassigned within its scope. On the other hand, const is used for variables that shouldn't change once they are set. It also has block scope like let, but it ensures that the value cannot be modified after initialization. These three options offer different levels of control and flexibility when working with variables in JavaScript.
Tell us the use cases of null and undefined.

In JavaScript, the values null and undefined have distinct roles. null is specifically used to indicate the deliberate absence of a value, typically when you want to convey that a variable should be empty or reset to an empty state on purpose. undefined signifies that a variable has been declared but hasn't been given any value, it functions as the default state for function parameters when they haven't been provided or when you try to access an object property that doesn't exist. Recognizing these differences is essential for precise variable handling and robust JavaScript coding practices.
What do you mean by REST API?

A REST API is like a set of rules for web applications to talk to each other. It uses standard commands like "get" (to ask for data), "post" (to send data), "put" (to update data), and "delete" (to remove data) to make requests to a web server. It's a way for different programs and websites to share information and work together on the internet.
